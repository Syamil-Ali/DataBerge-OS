🧠 Data-Berge OS — Intent Routing Flowcharts

How the new LLM-based routing works vs the old keyword matching

Architecture Comparison

LayerOld (Keyword Matching)New (LLM Intent Router)
Coordinator Keyword match → analyst or engineer LLM → analyst or engineer (unchanged)
Skill Routing Keyword match → intake/profiling/query/viz/reporting LLM intent router → intake/profiling/query/viz/reporting
QuerySkill LLM planner → profile/sql/clarify LLM planner → profile/sql/clarify (unchanged)
VisualizationSkill Keyword match only (no LLM) Keyword match + LLM planner for context understanding

Example 1: Analytical Question → QuerySkill

User
"How many loans were approved?"
1
API Endpoint
POST /projects/{id}/chat receives the message
chat_workflow.py loads dataset + last 10 history messages
2
Coordinator
LLM decides: DataAnalystAgent
"Question is analytical, best handled through profile reasoning or SQL"
3
DataAnalystAgent — Intent Router LLM
LLM receives: user message + history + profile summary
Returns: {"skill": "query", "rationale": "User wants a count of approved loans"}
4
QuerySkill — Analysis Planner LLM
LLM receives: full profile context + compact history + user question
Returns: {"mode": "sql", "sql": "SELECT COUNT(*) FROM dataset WHERE ...", "confidence": 0.85}
5
SQL Execution
DuckDB executes the query against the dataset
Result: [{"count": 12450}]
6
Response
Answer + SQL + evidence logged to MLflow (with full profile context)
12,450 loans were approved out of 24,000 total (51.9% approval rate).

Example 2: Chart Type Change → VisualizationSkill

Previous turn
"Can you generate a chart for approval vs rejected?" → Donut chart shown
User
"Give me bar chart"
1
Coordinator → DataAnalystAgent
Routes to analyst (same as before)
2
Intent Router LLM
LLM sees conversation history: "previous chart was donut of Approval"
Returns: {"skill": "visualization", "rationale": "User wants to change chart type of previous chart"}
3
VisualizationSkill — Keyword Match
_profile_chart() → No column matches "give me bar chart"
Falls through to history lookup
4
History Lookup
Finds previous payload: data = [{"label": "Approved", "count": 12450}, {"label": "Rejected", "count": 11550}]
suggest() creates bar chart from this data
5
LLM Chart Planner (fallback if suggest fails) LLM
If suggest() returned None, the LLM planner would interpret: "retype_previous to bar"
Returns: {"action": "retype_previous", "chart_type": "bar", "x_key": "label", "y_keys": ["count"]}
6
Response
Bar chart of Approval/Rejected with correct data ✓
Instead of the old bug showing "Distribution of Income" histogram

Example 3: Ambiguous Question → ProfilingSkill

User
"What stands out in this data?"
1
Intent Router LLM
LLM understands: user wants dataset overview, not a specific calculation
Returns: {"skill": "profiling", "rationale": "User wants to understand what's interesting about the dataset"}
2
ProfilingSkill
Returns dataset overview: structure, quality flags, key statistics
Deterministic — no LLM needed here

Old behavior: Keyword match for "stands out" → profiling (worked, but brittle)
New behavior: LLM understands intent → profiling (works even for "tell me about this data" which the old system would route to query)

Example 4: Report Request → ReportingSkill

User
"Write me a summary report for my boss"
1
Intent Router LLM
LLM understands: user wants a written document, not a chart or query
Returns: {"skill": "reporting", "rationale": "User wants a summary report document"}
2
ReportingSkill
Generates report: executive summary, key findings, recommendations
Deterministic template-based output
