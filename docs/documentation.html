Data Berge OS

A full-stack AI-powered data analytics platform for profiling, exploring, and reporting on structured datasets — with multi-table relational schema support and agentic AI workflows.

Overview Architecture Frontend Backend Workflow AI Agents Features Database Configuration API Reference

1. Overview Introduction

Data Berge OS is a self-contained analytics workspace that lets users upload structured data (CSV, Excel), automatically profile it, explore it through natural language chat powered by AI agents, and generate executive-level reports — all within a single application.

Data Pulse

Automated dataset profiling with per-column statistics, distributions, quality flags, bivariate analysis, and data engineering readiness scoring.

Explorer

Natural language chat interface with AI agents that generate SQL queries, charts, and insights against your data. Multi-session, multi-turn conversations.

Executive Report

Automated five-step report pipeline: readiness assessment, data investigation, narrative generation, governance review, and approval workflow.

Relational Schema

Multi-table Excel upload with automatic relationship inference, data dictionary mapping, and joined working dataset materialization.

OpenDOSM

Built-in connector to Malaysia's Open Data portal — browse and connect official datasets (CPI, population, demographics) with automatic schema detection.

AI Agents

Four-agent system with a model-led Team Manager, Data Analyst, Data Engineer, and Report Agent backed by bounded analytics tools and persistent memory.

Services

profiling.py (445 lines)

The core profiler. Takes a DataFrame and produces a comprehensive profile with per-column statistics (mean, median, std, quartiles, histograms, top values, word frequencies), metadata about column types, quality flags (missing cells, duplicates, empty datasets), Pearson correlations, and full bivariate analysis (Pearson for numeric-numeric, chi-square for categorical-categorical, ANOVA for numeric-categorical). Uses Polars when available, falls back to Pandas. Optionally uses SciPy for statistical tests.

query_engine.py

Builds and executes safe SELECT-only DuckDB queries. The build_sql function uses keyword matching to translate natural language into SQL patterns (row count, missing values, aggregations, correlations, group-by, outcome comparisons). The validate_select function blocks non-SELECT statements. Queries run against an in-memory DuckDB instance with the dataset registered as the dataset table.

relational.py (714 lines)

Multi-table ingestion engine. Loads multi-sheet Excel files (each sheet becomes a table), detects Data Dictionary sheets, infers foreign-key relationships via four methods: explicit PK/FK annotations, normalized column name matching, table-name FK patterns, and value coverage analysis. Builds per-table profiles and produces a unified relational schema.

data_engineering.py

Infers engineering roles for each column (identifier, measure, category, outcome, narrative, time), detects null-like markers, identifies datetime candidates, and computes a readiness score (0-10) penalizing missing cells, duplicates, and undocumented columns.

opendosm.py

Connector to Malaysia's OpenDOSM API. Fetches datasets (CPI, population, births, deaths, marriages, fertility, household profiles), scrapes metadata from the catalogue HTML, and builds relational schemas with automatic lookup table resolution from metadata field descriptions.

llm_observability.py

Comprehensive MLflow integration. Logs profile runs, chat runs with full execution traces (agent mode, active skill, SQL executed, chart generated, plan steps, debate rounds, prompt templates, token usage, cost data).

6. AI Agents Agentic System

Agent Architecture

The agent system has two layers: four product agents in app/agents/ and reusable, bounded skills in data_berge_core/skills/.

TeamManagerAgent

Owns each conversation turn. Gemini decides whether to reply directly or issue typed assignments to one or more specialists. It never answers factual dataset questions without delegating them to an agent with trusted tools.

DataAnalystAgent

Owns dataset intake, profiling, calculations, SQL, interpretation, and visualization. Manager-selected skills avoid a second routing call, while deterministic profile and query tools keep factual answers grounded.

DataEngineerAgent

Data preparation agent. Uses EngineeringSkill. Handles questions about data quality, schema meaning, readiness scoring, cleaning recommendations, semantic role inference, and typing issues.

ReportAgent

Owns report planning, narrative hierarchy, section selection, chart placement, stateful plan revision, and confirmed execution. Custom reports expose their complete plan before any report content is generated.

Manager Decisions

ActionWhen UsedBehavior
respondConversation or clarificationThe Team Manager replies directly without invoking a dataset tool.
delegateDataset or report workThe Team Manager assigns a typed task and skill to the appropriate specialist.
multi-agent delegateDistinct cross-domain workThe Team Manager assigns scoped tasks to multiple specialists and their grounded outputs are combined.

Agentic Foundation

Typed Assignments

Every manager decision is validated against a Pydantic schema containing an action, intent, focus, specialist, task, and allowed skill. Invalid decisions fall back to a single safe role assignment.

AgentMemory

JSON-file-backed persistent memory. Stores analyses, patterns, preferences, and summaries with TTL support and LRU eviction (max 500 entries). Injected into agent prompts as context.

Chat Flow (End-to-End)

  1. Frontend sends message with dataset_id and session_id to /api/projects/{id}/chat
  2. chat_workflow.run_chat_workflow resolves the dataset (handling relational schemas, materialization)
  3. Creates an AnalyticsTeam with the Team Manager and three specialist agents
  4. team.coordinator.respond() asks the Team Manager for a typed decision:
    • Replies directly for conversation, frustration, or clarification
    • Delegates dataset work to Analyst or Engineer and report work to Reporter
    • Allows the selected specialist to execute its bounded profile, SQL, chart, engineering, or report tools
    • Preserves structured artifacts such as SQL, rows, charts, and versioned report plans
    • Records insights to AgentMemory
    • Returns response with evidence, SQL, data, charts, metadata

LLM Integration

When AGNO_MODEL and a valid API key are configured, the Team Manager uses the LLM for autonomous assignment and specialist agents use it for SQL planning, interpretation, report narration, and visualization planning. Deterministic tools remain responsible for data access and validation. If the manager model is unavailable, the coordinator falls back to one safe specialist assignment. OpenAI-compatible endpoints are supported through AGNO_BASE_URL.

8. Database SQLite

SQLite database at data/app.db with 6 core tables. All flexible data is stored as JSON blobs in *_json columns. Per-user data isolation enforced by user_id columns on every table.

TableKey ColumnsPurpose
users id, email (unique), name, password_hash, storage_used Authentication, storage tracking
projects id, user_id, name, description Top-level workspace containers
datasets id, project_id, name, source_path, working_path, row_count, column_count, profile_json Uploaded data files with full profiling results stored as JSON
chat_sessions id, project_id, dataset_id, title Chat session containers
chat_messages id, session_id, role, content, payload_json Individual messages with agent metadata stored as JSON
artifacts id, dataset_id, kind (dashboard/chart/report), status (draft/approved/rejected), payload_json Generated outputs requiring approval
relational_schemas id, project_id, name, schema_json, status (draft/confirmed/active) Multi-table schema definitions with relationship metadata

Migration: The database layer uses a _migrate function that adds missing columns and tables to existing databases, enabling schema evolution without manual SQL migrations.
